# Live Testing Feedback — 2026-03-10

Second testing session. Display/eval separation refactor validated.

## Bugs Fixed This Session

- **Description not rendered**: Currency/gem description text (`ResolvedItem.description`) wasn't displayed in overlay. Added `.item-description` section.
- **Wrong tooltip colors**: Description, flavor text, and property colors didn't match in-game. Fixed: description `rgb(135,135,254)`, flavor/properties `rgb(127,127,127)`.
- **Property value styling**: Game renders label in gray, value in white. Was all gray. Split into `.prop-label` + `.prop-value`.
- **Header font**: Was FrizQuadrata primary. Game uses Fontin SmallCaps (natural small caps effect where first letter is taller). Swapped priority.
- **Empty sections rendered**: Sockets/item-level section rendered empty div for currency. Made conditional.
- **Trailing separator**: Extra separator line after description when nothing follows. Removed.
- **Currency/Gem sprites not wired**: Header and separator sprites were extracted but not used. Added imports + lookup entries.

### 2. Pseudo (sum) stats (poe-eval)
Like poe.trade's pseudo mods — `pseudo total life > 140` sums all matching stat lines across mods. New predicate type `PseudoStatValue`. The term "pseudo" is established PoE community convention.

### 3. Item browser for rule building (app)
When building rules, users need to look up base types, mods, etc. We have all this data in poe-data — could render a browsable database. Useful for Base Type field in rules.

### 4. HasStatText deprecation (poe-eval)
`HasStatText` uses text `contains` matching. Deprecate in favor of `HasStatId`. No new rules should use text matching.
